📤 Video Upload Flow
1
Admin picks MP4 → browser calls POST /api/admin/media/presign → server returns S3 presigned PUT URL + UUID filename
2
Browser PUT file directly to S3 with XHR (shows % progress) — bypasses Cloudflare entirely, no 100 MB limit
3
Browser POST /api/admin/media with title + presigned_filename → Flask saves DB record (hls_status = none)
4
Flask spawns daemon thread → video_processor.trigger() → downloads MP4 from S3 to /tmp
5
ffprobe detects resolution; ffmpeg encodes 360p / 720p / 1080p HLS (6-second segments, preset fast) + extracts thumbnail @ 2s
6
HLS tree + thumbnail uploaded to S3 hls/{id}/; DB updated: hls_path, duration_sec, thumbnail_filename, hls_status = done
▶ Video Playback Flow
1
Visitor opens page → GET /api/videos returns list with hls_url (S3 master.m3u8) + url (direct MP4)
2
Click video card → hls.js loads master.m3u8 from S3, selects quality by measuring download speed
3
Browser fetches .ts segments directly from S3 (CORS allows GET *); switches 360p ↔ 720p adaptively mid-playback
4
URL updates to #video-{id} — shareable deep link; copy-link button in modal; Safari uses native HLS; fallback to direct MP4
